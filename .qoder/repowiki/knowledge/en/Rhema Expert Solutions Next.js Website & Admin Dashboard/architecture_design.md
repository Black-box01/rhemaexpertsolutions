Monorepo-style Next.js App Router project. The `app/` directory is the routing layer: public pages live under `app/` (home, coding-classes, competition, professional-trainings, verification-success) and the protected admin UI under `app/admin/dashboard/page.tsx`. Server-side mutations are isolated in `app/actions/*.ts` (`admin`, `auth`, `notes`, `registration`, `coding-classes`) marked `'use server'`; they call `supabaseAdmin` from `lib/supabase-admin.ts` (service-role client that bypasses RLS) while read-only clients use `lib/supabase.ts`. Authentication is cookie-based via `rhema_admin_auth` set by `login()` in `app/actions/auth.ts`, checked by every action through `checkAuth()`. The admin dashboard is a single large React client component that switches tabs to render CRUD forms per entity (services, clients, team, competitions, newsletter, settings, registrations, coding-classes, staff-notes, professional-trainings). E-notes add a dedicated tab with pagination, ILIKE search, status/category/priority filters, drag-and-drop file upload to the `staff-notes` Supabase Storage bucket, and an email notification via `nodemailer` (`sendENoteNotificationEmail`). Shared UI lives in `components/` (Header, HeroSlideshow, ImageGrid, etc.) and static assets in `public/img/`. Database schema and migrations are declared as SQL files at the repo root (`supabase_schema.sql`, `supabase_migration_*.sql`), with TypeScript types generated into `types/supabase.ts`.
