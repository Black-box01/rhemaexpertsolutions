Monorepo-style Next.js App Router project. `app/` holds the public pages (`page.tsx`, `coding-classes`, `professional-trainings`, `competition`, `verification-success`) plus an `admin/` sub-app whose single `dashboard/page.tsx` is a large client-side tabbed UI (services, clients, team, competitions, newsletter, settings, registrations, coding-classes, staff-notes, professional-trainings). Server actions in `app/actions/` (`admin.ts`, `auth.ts`, `notes.ts`, `registration.ts`, `coding-classes.ts`) are the only data boundary — they call Supabase via two clients: a public anon client (`lib/supabase.ts`) for read-only access and a service-role client (`lib/supabase-admin.ts`) used by server actions to bypass RLS. `types/supabase.ts` declares all table shapes (e.g. `RhemaStaffNote`, `RhemaRegistration`, `RhemaCodingClassRegistration`). The e-note subsystem lives in `app/actions/notes.ts` + `components/Header.tsx` navigation; file attachments go to the `staff-notes` Supabase Storage bucket under `attachments/`. Email notifications on note creation flow through `lib/email.ts` using nodemailer. Static assets live under `public/img/` grouped by course category. Build/dev scripts are standard Next (`next dev`, `next build`, `next start`).
