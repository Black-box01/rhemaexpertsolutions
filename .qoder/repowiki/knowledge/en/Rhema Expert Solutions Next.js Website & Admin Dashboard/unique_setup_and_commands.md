Requires `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` env vars; the admin client falls back to the anon key when the service role key is missing but writes will fail if RLS is enabled. Database schema and migrations are plain SQL files at the repo root (`supabase_schema.sql`, `supabase_migration_*.sql`) applied directly to the Supabase project rather than via a CLI pipeline.
